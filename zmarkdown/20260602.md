### MyFlow 当前架构

当前 velocity field 是三部分叠加：

```text
base MyFlow velocity
  + GO response prior residual
  + perturbation-level STRING/PPI graph residual
```

其中：

```text
base MyFlow:
  control cell x_t + time t + condition embedding
  -> velocity

GO response prior:
  output genes + perturbation genes in one combined symbol table
  TxPert GO graph over combined genes
  graph message passing
  pair(output gene z_i, perturbation gene z_p)
  -> gene-wise rho
  -> residual added to velocity

Perturbation graph prior:
  perturbation genes only
  TxPert STRING/PPI graph over perturbation genes
  graph message passing
  pair(output gene z_i, graph-encoded perturbation z_p)
  -> gene-wise rho_pert
  -> residual added to velocity
```

Important distinction:

```text
TxPert:
  perturbation graph encoder -> endpoint / response predictor

MyFlow:
  perturbation graph encoder -> velocity residual
  keep flow/distribution modeling as main generator
```

So the design is not to turn MyFlow into a TxPert endpoint model. It adds TxPert-like perturbation-specific reasoning while preserving MyFlow's distribution-generation path.
